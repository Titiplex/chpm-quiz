# Non-technical user manuals

These manuals explain normal platform tasks without assuming software-development knowledge. Your organization may use different names, URLs, contacts, or approval steps; local policy takes precedence where it is stricter.

## Essential safety rules for everyone

1. Use your own named account. Never share a staff login.
2. Never paste respondent links, terminal tokens, temporary passwords, encryption keys, or personal data into email, chat, tickets, or screenshots unless an approved procedure explicitly requires it.
3. Do not try to identify a respondent from a public code, timestamps, building, free text, or small statistical groups.
4. Use the minimum information necessary. Ask respondents not to enter names, email addresses, phone numbers, exact addresses, or exact dates in free-text answers.
5. Lock your workstation and sign out when finished, especially on shared clinical or research workstations.
6. Stop and report unexpected access, wrong-scope data, repeated errors, lost devices, suspicious links, or accidental disclosure.
7. A successful screen action is not permission to use the data for a new purpose. Follow the approved study/service purpose and retention rules.

## Common terms

| Term | Meaning |
| --- | --- |
| Questionnaire | The overall instrument, identified by a stable code |
| Version | A dated/labeled structure and language of a questionnaire |
| Draft | Editable version that respondents cannot normally use |
| Published | Validated, immutable version eligible for invitations |
| Invitation | One tracked request for a person to complete one published version |
| Public code | Pseudonymous operational identifier; not an authentication secret by itself |
| Respondent link/token | Secret link that grants access to one response session |
| Terminal | Managed on-site device used to open assigned invitations |
| Submission | Final locked response |
| Identity vault | Separate encrypted store for direct contact data |
| Suppression threshold | Minimum group size below which granular statistics are hidden |

## Getting help

- For a normal workflow question, contact your site manager or project administrator.
- For a technical outage or device issue, contact the technical operations owner.
- For privacy rights, inappropriate data access, or identity questions, contact the DPO/privacy owner.
- For a possible incident, follow the [incident response procedure](../production/incident-response.md) immediately. Do not investigate by exporting more data.
